# 搭子地图 - 需求确认文档

> 版本：v1.0  
> 日期：2026-04-14  
> 定位：功能展示级项目，非生产级部署

---

## 一、项目概述

**项目名称**：搭子地图（Partner Map）

**目标用户**：大学生群体

**核心场景**：用户在App上发布搭子需求（附带定位），以地图形式展示，其他用户可点击进入私聊，由发起者审批确认参与。完成互评后形成个人信用档案。

**技术栈**：
- 客户端：Android 原生 Java
- 地图SDK：高德地图
- 后端：待设计

## 二、用户体系

### 2.1 注册与登录
- 开放注册，无需edu邮箱或学号认证
- 无需实名信息，昵称即可使用
- 注册时须从预设学校列表中选择一所关联学校（一人一校）

### 2.2 个人资料页
展示以下信息：
- 头像
- 昵称
- 关联学校
- 发布历史（列表）
- 参与历史（列表）
- 赴约率（百分比）
- 好评率（百分比）

## 三、需求发布

### 3.1 发布字段

| 字段 | 说明 | 必填 |
|------|------|------|
| 标题 | 需求标题 | 是 |
| 描述 | 需求详情 | 是 |
| 分类 | 学习/运动/美食/出行/娱乐/购物 | 是 |
| 需求位置 | 经纬度坐标（可与发布位置不同） | 是 |
| 发布位置 | 用户发布时的定位（可与需求位置不同） | 是 |
| 人数上限 | 发起者自定义 | 是 |
| 预定时间 | 具体到场的日期与时间 | 是 |
| 性别要求 | 限制参与者的性别 | 否 |
| 消费说明 | 费用分摊或预估消费 | 否 |

### 3.2 编辑与快照
- 发布后可编辑需求内容
- 用户点击参与时，系统保存该时刻的需求详情快照
- 已参与用户看到的是自己参与时的快照版本
- 需求主页展示最新版本

### 3.3 删除与取消
- 发起者可主动删除需求
- 发起者可主动取消需求

## 四、需求状态生命周期

### 4.1 三态模型

```
招募中 → 已满员 → 已结束
```

| 状态 | 说明 |
|------|------|
| 招募中 | 需求可见，可进入聊天室申请参与 |
| 已满员 | 参与人数达到上限，不可再进入聊天室 |
| 已结束 | 需求完成，进入评价阶段 |

### 4.2 失效机制
- 发布者设定失效时间：预定时间之前N分钟自动失效（变为已结束状态）
- 若未满员且到达预定时间，需求自动标注已结束

## 五、地图与列表

### 5.1 地图展示
- 使用高德地图SDK展示需求点位
- 每个PIN点按分类显示不同图标（6个分类对应6种图标）
- 点击PIN点可查看需求摘要，进一步可点击进入详情

### 5.2 列表视图
- 地图与列表双视图切换
- 列表按时间倒序展示需求

### 5.3 筛选与过滤
- 按距离筛选
- 按时间筛选
- 按学校筛选（全部学校 / 仅本校）
- 按分类筛选

## 六、聊天与参与流程

### 6.1 私聊机制
- 用户点击需求详情后，可进入与发起者的1对1私聊室
- 进入聊天室即可直接发消息
- 满员状态下不可进入聊天室

### 6.2 审批流程
- 发起者在聊天室内或管理页面审批参与请求
- 发起者确认后，用户正式加入该搭子

### 6.3 聊天室生命周期
- 评价完成后，聊天室自动解散
- 历史聊天记录不保留

---

## 七、评价体系

### 7.1 评价维度
- **赴约率**：实际赴约次数 / 应赴约次数（百分比）
- **好评率**：收到好评次数 / 总被评次数（百分比）

### 7.2 评价规则
- 双方互评：发起者评参与者，参与者评发起者
- 参与者之间也可互相评价
- 每次评价为二元评价：赴约/未赴约 + 好评/差评
- 不展示具体评价内容和评价者身份
- 仅在个人页展示赴约率和好评率两个数值

### 7.3 评价触发
- 方式一：发起者手动标记需求为"已完成"
- 方式二：预定时间24小时后自动触发评价流程
- 评价期限：10天内完成评价，超时默认好评赴约

### 7.4 赴约判定
- 未赴约仅记录并反映在赴约率中，无其他惩罚

## 九、数据模型概要

| 字段 | 类型 | 说明 |
|------|------|------|

| schoolId | String | 关联学校ID |

| praiseRate | Float | 好评率 |

### Request（搭子需求）
| 字段 | 类型 | 说明 |
|------|------|------|
| requestId | String | 需求ID |

| title | String | 标题 |
| description | String | 描述 |

| requestLng | Double | 需求位置经度 |

| publishLng | Double | 发布位置经度 |
| maxParticipants | Int | 人数上限 |
| scheduledTime | Long | 预定时间 |
