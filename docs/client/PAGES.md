## 三、核心交互流程

### 3.1 发布需求 → 地图展示

```
用户点击FAB → 打开发布页 → 填写表单
  ├─ 选择分类（6选1）
  ├─ 在地图上选点作为需求位置
  ├─ 自动获取当前位置作为发布位置
  ├─ 设定预定时间与失效提前量
  └─ 提交 → 服务端创建需求
       ├─ 状态设为"招募中"
       └─ 返回首页，地图新增PIN点
```

### 3.2 参与搭子 → 聊天

```
点击PIN点/列表项 → 需求详情页
  ├─ 点击"私信发起者"
  │    ├─ 已有聊天室 → 直接进入
  │    └─ 新聊天室 → 调用API创建 → 进入聊天页
  └─ 满员状态 → 显示"已满员"
```

### 3.3 审批 → 加入

```
发起者在聊天页看到申请者消息
  └─ 点击"同意加入"
       ├─ 创建参与记录
       ├─ 保存需求快照
       ├─ 通知申请者
       └─ 若达到人数上限 → 需求状态变为"已满员"
```

### 3.4 评价流程

```
需求完成（发起者标记 或 预定时间+24h自动）
  → 服务端为所有人生成待评价记录
  → 用户收到通知 → 进入评价页
  → 逐一对参与者评价：赴约？+ 好评？
  → 10天超时自动好评+赴约
  → 全部完成后更新参与各方 attendRate / praiseRate
  → 聊天室解散
```
